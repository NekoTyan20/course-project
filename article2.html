У чому різниця між адаптивним дизайном та Mobile First?

12 апреля 2023, 14:15

Адаптивний веб-дизайн (Adaptive Web Design, AWD)

При проектуванні веб-сайту в адаптивному дизайні розробляється кілька статичних макетів для різних типів пристроїв. Шаблони містять структуру розміщення елементів, залежно від ширини дисплея. Зазвичай макети створюють для декількох стандартних величин ширини, наприклад: 320 px, 480 px, 768 px, 960 px, 1200 px, 1600px. Скрипти визначають розміри екрана користувача та завантажують відповідний шаблон на основі цих контрольних точок. В адаптивних макетах враховуються особливості різних пристроїв. Наприклад, для десктопних екранів – лише горизонтальна орієнтація та велика область перегляду, а для планшетів та смартфонів – сенсорне керування та можливість повороту девайсу для перегляду.

Адаптивний веб-дизайн використовує медіазапити для зміни стилів веб-сторінки в залежності від розміру екрану. Наприклад, для екранів з розміром менше 768 пікселів можна використовувати інший стиль, ніж для більш великих екранів. Для цього в CSS використовуються медіа-запити, наприклад:

Приклад:


/* Стилі для екранів з розміром до 768 пікселів */
@media (max-width: 768px) {
  body {
    font-size: 16px;
  }
}

/* Стилі для екранів з розміром від 768 пікселів і більше */
@media (min-width: 768px) {
  body {
    font-size: 18px;
  }
}

Mobile First також використовує адаптивний дизайн, але починає з проектування для маленьких екранів і потім розширює дизайн для більших екранів. Наприклад, в Mobile First стилі для екранів з розміром менше 768 пікселів будуть описані в основному CSS-файлі, а стилі для більших екранів - у відповідних медіа-запитах. Ось приклад коду, що використовує Mobile First:

Приклад:

/* Основні стилі для маленьких екранів */
body {
  font-size: 16px;
}

/* Стилі для екранів з розміром від 768 пікселів і більше */
@media (min-width: 768px) {
  body {
    font-size: 18px;
  }
}

Чуйний дизайн (Responsive Web Design, RWD)

У цьому випадку створюється один гнучкий макет, який дозволяє сторінці підлаштовуватися під будь-яку ширину екрану. Автоматична зміна розмірів контенту працює на всіх пристроях за рахунок рухомої сітки, гнучких зображень та медіазапитів.

Responsive Design добрий тим, що не потрібно розробляти кілька версій сайту. У той же час необхідно ретельно перевіряти, чи правильно відображаються всі елементи на екранах різних розмірів.

Для прикладу коду, який може допомогти зробити веб-сайт чуйним до розміру екрану, можна використати CSS медіазапити. Медіазапити дозволяють змінювати стилі для елементів на основі розміру екрану або інших характеристик пристрою.

Ось приклад коду, який змінює розмір шрифту для елементів з класом "title" для екранів менше 768 пікселів:

Mobile First Design

При такому підході спочатку розробляється та завантажується версія для мобільних пристроїв із невеликим екраном. Потім вона масштабується і залежно від платформи або браузера розробники можуть додати додаткові опції або інші стилі. Таким чином, з якого б пристрою людина не переглядала сайт - він завжди отримає версію, що коректно відображається.

Приклад:


@media only screen and (max-width: 768px) {
  .title {
    font-size: 16px;
  }
}

Тут ми використовуємо медіазапити, щоб вказати, що стилі в межах цього блоку будуть застосовуватися тільки для екранів з максимальною шириною 768 пікселів. Якщо екран менше 768 пікселів, шрифт для елементів з класом "title" буде 16 пікселів, а не той, який вказаний за замовчуванням.

Це тільки один з багатьох прикладів того, як можна використовувати медіазапити, щоб зробити веб-сайт чуйним до розміру екрану. Існує багато інших способів, таких як використання гнучких зображень та рухомих сіток, які допомагають забезпечити добру роботу сайту на пристроях з різними розмірами екрану.
